JavaScript Objects vs. Ruby Hashes

Not so different after all

May 2nd 2015

Ruby and JavaScript, both object oriented web programming languages, have their own ways of storing key/value pair data sets. In Ruby, key/value data structures are called "hashes," and in JaveScript, they are called "Objects." Let's check out an example of the same data structure written in both languages so we can see the differences in each language. First, check out this ruby hash that stores the information about an athlete.

As you can see, we are simply able to set a number of key/value pairs for the athlete, Kelly Slater. The syntax for creating this hash is simply hash_name = {key: value, key2: value2, etc.}. now lets look at this same data in the context of a JavaScript object.

So lets break down the main differences. At a quick glance, they don't look all that different! In order to define the object variable in JavaScript, we must include var = at the bedinning of the object definition. You then see at the end of the object, we much close the object with a semicolon. These are the two main differences of Ruby vs. JavaScript in the syntax. Now, the next main differences are how we can call values. The syntax for calling Kelly Slater's name in JavaScript is kellySlater.name, whereas in Ruby it is kelly_slater[:name]. Overall, the two languages approach these types of data sets in a similar manner. Ruby Hashes and JavaScript Objects aren't that different after all!
