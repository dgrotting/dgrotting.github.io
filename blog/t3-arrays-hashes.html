Arrays vs. Hashes

What's the difference?

April 2nd 2015

An Array...A Hash? What the heck are those? Well let me tell you. It's actually pretty simple. In Ruby, Arrays and hashes are places to store information. They are both collecitons of objects stored in different ways. In short, an array stores objects in a specific order by integer index. That means that any of the objects can be called by their order in the 'line.' Pretty easy, huh? A hash on the other hand is a list of stored 'key-value' pairs stored in a non sequential order. That means that the values can be called by their relative keys, and vice versa. Let me give you some examples of each below to further your understanding.

Arrays

I'll give you an example of an array. Say you have been putting together a list of your favorite candy bars. You can use an array to store this list of candy bars, even order them by your favorite! Obviously, Snickers are your favorite so they will go first in the array at index number '0'. (Don't get freaked out, arrays always start at the zeroth position first) You can then have the following 19 (or more if you have a sweet tooth!) favorite candy bars to take up the sequential index position. If you ever forget what your 10th favorite candy bar is, you can simply call it in the array. You would simply use the ruby code: candy_bars[9] and your 10th favorite candy bar will appear! Below is what your array would look like in ruby.

candy_bars["Snickers", "Milky Way", "Reeses", "Almond Joy", "100 Grand", "Baby Ruth", "Butterfinger", "Dove", "And many more!"]

Hashes

Now a hash on the other hand is a key-value pairing. So how is this different than an array? Well, an array object can be accessed by its index number. On the other hand, a hash value can only be accessed by it's relevant key. This means that the hash objects are not stored in order, but each hash value always hangs out with it's key. So - you could use hashes to give a rating descriptor to each of your candy bars! The rating could be the key and the value could be the value. Check out my new hash. I bet it will make sense to you! You can easily call the candy bar by the rating system that I have put on it. (You may also notice that we use a different bracket system for hashes)

candy_bars{"delicious" => Snickers", "tasty" => "Milky Way", "flavorful" => "Reeses", "gross" => "Almond Joy", "pretty good" => "100 Grand", "underrated" => "Baby Ruth", "over rated" => "Butterfinger", "over priced" => "Dove", "everything else!" => "And many more!"}

See, that wasn't so bad! Was it?
